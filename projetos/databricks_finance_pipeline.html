Databricks Finance Pipeline

Pipeline end-to-end consumindo dados reais de ações bancárias, fundamentado na Arquitetura Medalhão.

O Databricks Finance Pipeline é um projeto de engenharia de dados focado em ingestão, processamento e modelagem analítica estruturada. Ele consome o histórico de 2 anos de ações dos maiores bancos (JPM, BAC, GS, C, WFC) através da API yfinance, processando todo o ciclo de vida do dado dentro do ecossistema Databricks.

Arquitetura Medalhão

O fluxo de dados foi desenhado para isolar responsabilidades, garantindo que o dado bruto seja preservado enquanto as camadas subsequentes aplicam regras de qualidade e negócio:

yfinance API → Bronze (Raw) → Silver (Clean) → Gold (KPIs) → Dashboard
  • Camada Bronze (Ingestão): Download e armazenamento dos dados brutos exatamente como fornecidos pela fonte, servindo como histórico imutável.
  • Camada Silver (Transformação): Limpeza, casting de tipos e validação de dados utilizando o poder de processamento distribuído do PySpark.
  • Camada Gold (Negócios): Engenharia de features e métricas financeiras consolidadas, incluindo cálculo de percentual de retorno diário, média móvel de 7 dias e variação de faixa de preço.

Estrutura de Notebooks

A orquestração local (Databricks Free Edition) é dividida nas seguintes rotinas independentes:

Notebooks/
├── 01_bronze_ingest.py    # Extração de dados via yfinance e persistência
├── 02_silver_clean.py     # Data cleaning e schema validation
├── 03_gold_transform.py   # Cálculos de KPIs financeiros
└── 04_analyze.py          # Queries analíticas (Databricks SQL) e Dashboard

Camada de Consumo

A ponta final do pipeline entrega os dados modelados (Delta Tables) diretamente para o Databricks SQL, viabilizando a visualização de tendências do mercado financeiro por meio de dashboards integrados.
