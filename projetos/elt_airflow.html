ELT Airflow Pipeline

Orquestração de extração via API REST, carga em banco relacional e modelagem de dados utilizando dbt.

Este projeto consolida um fluxo completo de Engenharia de Dados focado no paradigma ELT (Extract, Load, Transform). O diferencial desta arquitetura é a integração de ponta a ponta com um Data Source proprietário (desenvolvido no projeto Hyper-Rent), demonstrando domínio tanto na engenharia de software que serve o dado, quanto na infraestrutura analítica que o consome.

Arquitetura de Redes Isoladas (Docker)

Para simular um ambiente de produção real e seguro, a infraestrutura foi dividida em duas redes Docker distintas que se comunicam via proxy:

  • Rede do Data Source: Hospeda a API REST (Node.js/Express) protegida por um Nginx operando como Forward Proxy.
  • Rede do Data Pipeline (Airflow): Contém os workers de extração, o banco de dados analítico e o motor de transformação.

O Fluxo ELT

1. Extract (Python)

O Apache Airflow orquestra um script em Python responsável por bater no endpoint seguro da API (http://localhost/api/admin/cars?apikey=). O uso de chaves de API garante a extração de dados pré-analisados de forma governada, respeitando os contratos de dados da aplicação de origem.

2. Load (PostgreSQL)

Os dados brutos coletados em formato JSON são carregados diretamente, sem manipulações pesadas em memória, para a camada Raw/Staging de um banco de dados PostgreSQL isolado. Isso garante que o Data Lake/Warehouse tenha um histórico não corrompido do que foi ingerido.

3. Transform (dbt)

Com os dados já residindo no PostgreSQL, o Airflow dispara os modelos do dbt (Data Build Tool). O dbt executa o processamento pesado de limpeza, padronização e estruturação dimensional utilizando o motor de SQL nativo do próprio Postgres. Isso finaliza a transição do dado bruto para um modelo analítico pronto para consumo (Views/Marts).
