Weather ETL Pipeline

Automação de coleta, processamento e armazenamento de dados climáticos executada inteiramente em containers.

O Weather ETL Project foi desenvolvido para automatizar a coleta contínua de dados climáticos de diversas cidades e consolidá-los em um banco de dados relacional. Trata-se de um pipeline ETL (Extract, Transform, Load) completo, encapsulado nativamente dentro de containers Docker, eliminando a necessidade de qualquer instalação local de dependências além do próprio motor de virtualização.

Mecânica do Pipeline

  • Extract (Extração): O script lê uma lista de cidades pré-definidas a partir de um arquivo CSV e consome dados em tempo real através da WeatherAPI.
  • Transform (Transformação): Utilizando a biblioteca Pandas, os dados em formato JSON são limpos e padronizados. Os nomes das colunas são ajustados para os padrões SQL e um timestamp de coleta é injetado em cada registro para garantir a rastreabilidade temporal.
  • Load (Carga): Os dados processados são carregados e anexados à tabela final em um banco de dados PostgreSQL.

Detalhes Técnicos e Infraestrutura

O projeto foi desenhado focando em resiliência e independência de serviços:

  • Python: Motor central de todas as lógicas de requisição (requests) e modelagem de dados (Pandas).
  • Docker Compose: Orquestra a subida conjunta da aplicação (script) e do banco de dados como serviços isolados na mesma rede.
  • PostgreSQL: Camada de armazenamento que retém o histórico climático confiável para análises futuras.
  • Healthchecks (Docker): Ponto de segurança crucial. A execução do script em Python é programada para aguardar até que o container do PostgreSQL esteja com o status "Healthy" (totalmente pronto para receber conexões), prevenindo quebras e erros de timeout na inicialização.

Como Executar Localmente

1. Crie um arquivo .env na raiz do projeto com as seguintes variáveis de ambiente:

WEATHER_API_KEY=sua_chave_aqui
DATABASE_STRING=postgresql://postgres:vini@postgres:5432/etl
Nota: A chave de API pode ser gerada gratuitamente no portal da WeatherAPI. A string de conexão do banco já está ajustada para a rede do Docker, mas caso altere credenciais (como a senha), certifique-se de refletir as mudanças no arquivo docker_compose.yml.

2. Com o arquivo .env configurado, suba o ambiente completo executando:

docker compose -f Docker/docker_compose.yml up --build
